# Demo Checklist

This is the current recording plan for the final mobile-app demo.

## Before Recording

- Use a phone or emulator build, not web or desktop
- Keep the device screen large enough to read clearly in the recording
- Open Firebase Console before showing backend features
- Close unrelated windows, banners, and notifications
- Confirm audio is clear before starting

## Current App Demo Order

1. Cold-start the app from a clean launch
2. Show the Home screen
3. Navigate to Log Meal from the main CTA and bottom navigation
4. Return to Home and verify the bottom navigation still works
5. Show the Login screen
6. Trigger one validation error on Login
7. Show the Signup screen
8. Trigger one validation error on Signup
9. If Firebase auth is finished later, repeat the flow with real registration and login
10. If CRUD is finished later, record create, read, update, and delete while Firebase Console is visible

## Recording Rules

- No slide deck
- No team introductions
- Focus on app functionality only
- Keep the recording continuous if possible
- Aim for 10 to 15 minutes
